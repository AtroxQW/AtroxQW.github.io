顺序表及其基本操作的实现

¶顺序表

¶1. 顺序表的定义

线性表每个数据元素的存储位置都和线性表的起始位置相差一个和该数据元素的位序成正比的常数。因此，线性表种的任一数据元素都可以随机存取。所以线性表的顺序存储结构是一种随机存取的存储结构。通常用高级程序语言中的数组来描述线性表的顺序存储结构。

静态分配一维数组

静态分配时，由于数组的大小和空间事先已经固定，一旦空间占满，再加入新的数据就会产生溢出。因此，我们可以采用动态分配。

动态分配一维数组

采用动态分配的策略，我们可以在初始化顺序表的时候，进行分配：

L.data = (ElemType *) malloc (sizeof(ElemType) * InitSize)

¶3.顺序表的插入和删除

插入操作

往顺序表 L 第 i 个位置上插入指定元素 e。若 i 的输入不合法，则返回 false，表示插入失败；否则，将第 i 个元素及其后的所有元素依次向后移动一个位置，腾出一个空位置插入新元素 e ，顺序表长度加 1 ，返回 true。

  • 最好情况：新元素插入到表尾，不需要移动元素，时间复杂度 T(n) = O(1)

  • 最坏情况：新元素插入到表头，需要将原有的 length 个元素全部向后移动，时间复杂度 T(n) =O(n)

  • 平均情况：假设新元素插入任一位置概率位 1/n + 1，则平均循环次数为 np + (n -1)p + … + 1*p = n / 2。

因此，顺序表插入算法的平均时间复杂度为O(n)。

删除操作

删除顺序表 L 第 i 个位置上的元素，用引用变量 e 返回，若 i 的输入非法，返回false；否则，将被删除元素值赋给 e，将其后的元素向前移动一个位置，返回true。

  • 最好情况：删除表尾元素，无需移动元素，时间复杂度为O(1)

  • 最坏情况：删除表头元素，移动除表头外所有元素，时间复杂度O(n)

  • 平均情况：删除任一位置元素概率为1/n，则删除结点需要移动结点的平均次数：

    $\sum_{i-1}^n p_i(n-i) = \sum_{i=1}^n \frac{1}{n}(n-i) = \frac{1}{n} \frac{n(n-1)}{2} = \frac{n-1}{2}$

因此，顺序表删除算法的平均时间复杂度为O(n)。

¶4.顺序表的查找

顺序表的按位查找

数组静态分配和动态分配的顺序表按位查找的语句一样。

由于顺序表的各个数据元素在内存中连续存放，因此可以根据起始地址和数据元素大小立即找到第 i 个元素——“随机存取”，时间复杂度为 O(1)。

顺序表的按值查找

  • 最好情况：查找的元素就在表头，只需比较一次，复杂度O(1)

  • 最坏情况：查找的元素在表尾，需要比较 n 次，复杂度O(n)

  • 平均情况，假设查找元素在任一位置概率为1/n，平均比较次数为：

    $\sum_{i=1}np_i×i=\sum_{i=1}n\frac{1}{n}\frac{n(n+1)}{2}=\frac{n+1}{2}$

因此线性表按值查找算法的平均时间复杂度为 O(n)

题目：从有序顺序表中删除所有其值重复的元素，使表中所有元素的值均不同。

思路：因为是有序的顺序表，值相同的元素一定在连续的位置上。我们可以从头开始遍历，如果发现现在的元素和后一个元素值相等，则将现在元素之后的所有元素向前移动 1 位，达到删除现在这个元素的目的。

思路：

将数组R的首元素用 temp 保存，从尾元素开始，依次左移一位，最终令最后一位的值为 temp。

¶10.第十题

题目：一个长度为L(L >= 1)的升序序列 S ，处在第[L/2]（向上取整）个位置的数成为 S 的中位数。例如，若序列 S1 = （11， 13， 15，17， 19），则 S1 的中位数是 15，两个序列的中位数是含有它们所有元素升序序列的中位数。例如 S2 = （2， 4， 6， 8 ，20），则 S1 和 S2 的中位数是 11。现在有两个等长升序序列 A 和 B，试设计一个在时间和空间都尽可能高效的算法，找出两个序列 A 和 B 的中位数。

思路：若两个等长序列长度都为 length， 则要找的为第length个数是中位数。用 i 和 j 两个指针指向序列 A 和序列 B ，如果A[i] <= B[j]， i++，同时计数器+1，判断计数器是否等于length ，如果相等，则中位数是A[i - 1];否则 j++，同时计数器+1，做同样判断，如果满足条件，中位数位 B[j - 1]；

整体的思路类似于合并两个有序线性表。

¶12.第十二题

题目：给定一个含有 n 个整数的数组，请设计一个在时间上尽可能高效的算法，找出数组中未出现的最小正整数。例如，数组（-5 3 2 3）中未出现的最小正整数为 1，数组（1 2 3）中未出现的最小正整数为 4。

前言：对于一个长度为N的数组，其中没有出现的最小正整数只能在[1, N+1]中。这时因为如果[1, N]如果都出现了，那么答案是 N + 1，否则答案是[1, N] 中没有出现的最小正整数。

思路1：如果没有额外的时空复杂度要求，那么很容易实现。从最小的正整数 1 开始，依次判断2、3、4直到数组的长度 N 是否在数组中。如果当前考虑的数不在这个数组中，我们就找到了这个缺失的最小正整数；如果从1到N都在数组中，那么未出现的最小正整数为 N + 1。这个算法的时间复杂度位O(n^2)，空间复杂度为O(1)。

**思路2:**将数组视为哈希表。把 1 放在下标为 0 的位置， 把 2 放在 下标为 1 的位置上…。这个完成之后，遇到的第一个他的值不等于下标加1 的位置那个数就是我们缺失的那个数。

复杂度分析：

  • 时间复杂度：O(N)，这里 N 为数组的长度，需要使用均摊复杂度分析。
    • 这个while循环不会每一次都把数组里面的元素都看一遍。
    • 如果有一些元素在某一次的循环中被交换到了它们应该在的位置上，那么在后续的遍历中，由于它们已经在正确的位置上了，程序再看到它们的时候，就会被跳过。
    • 平均下来，每个数只需要看一次就可以了，while循环体被执行很多次的情况不会每次都发生。
  • 空间复杂度：O(1)

¶13.第十三题

题目：定义三元组(a, b, c)(其中 a b c 均为正数)的距离 D = |a - b| + |b - c| +|c - a|。给定三个非空整数集合 S1 、S2 和 S3 ，按升序分别存储在三个数组中。请设计一个尽可能高效的算法，计算并输出所有可能的三元组(a, b, c)(a∈S1， b∈S2， c∈S3)中的最小距离。例如 S1 = {-1 0 9}， S2 = {-25 -10 10 11}， S3 = {2 9 17 30 41}，则最小距离为2，相应的三元组为(9 10 9)。

思路：设置三个指针，分别用于遍历这三个数组。因为3个数组已经有序，我们的目的实际上是找i到一个尽可能接近的三元组。当前三元组为(a b c)，我们遍历使用的下标变量分别为 i j k。当a[i] < b[i] && a[i] < c[k]的时候，更新 i，其他两个下标变量更新同理。同时，对于每一组三个下标变量，我们计算其对应元素的距离值，然后与上一次的距离值相比较，并取其中较小的一个和作为新的距离值。
